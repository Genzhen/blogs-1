## 模块

### 模块化历史

* 文件模块化

不同的功能模块放在单独的js文件，一个script标签代表一个模块。

* 命名空间模块化

每一个模块包裹在一个全局对象下，模块之间实现不相互干扰。

* IIFE - 立即执行函数模块

用立即执行函数包裹实现私有空间，入参作为依赖声明。

* 规范化模块

AMD - Require.js、CMD - sea.js 

常见的模块实现, [参见 src目录](./src)

## 模块划分

* 模块划分的依据和原则

  - UI设计的建模

  - 后期的可维护性

  - 需求变化的可扩展性和可定制化

  - 尽量符合标准HTML规范
